# 2026-05-16 — 本地开发环境默认已打开规则回流

## 背景

NACP 日常测试时，用户通常已经手动打开本地前后端开发环境。

因此测试流程不应默认执行启动、重启、kill 进程或端口切换；应先探测并复用现有环境。

## 更新

1. 更新 CAF Stage A：
   - 默认本地前后端已由用户打开。
   - 测试只探测和使用。
   - 不可达时报告环境门禁失败，不擅自启动或重启。
2. 更新 API 驱动测试夹具规范：
   - 增加“本地开发环境状态”约定。
   - 增加本地环境探测规则。

## 固定规则

```text
默认复用用户已打开的本地开发环境
-> 探测 http://localhost:5173/api/status
-> 读取 server_address
-> 通过 API 执行测试
-> 环境不可达时报告，不自行启动/重启
```

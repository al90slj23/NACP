# 2026-05-16 — gogogo.sh 移除本地数据库并直连测试站数据库

## 操作内容

1. 取消 NACP 本地开发环境的本地 MySQL 方案，不再维护 `nacp-mysql-dev`。
2. 删除 `docker-compose.dev.yml`，避免误启动本地测试库。
3. 调整 `gogogo.sh`：
   - 本地前端固定 `23901`。
   - 本地后端固定 `23900`。
   - `./gogogo.sh 0 a`、`./gogogo.sh 0 c`、`./gogogo.sh 0 restart` 运行时从 `nacp.m.srl:/opt/nacp/.env` 读取 `SQL_DSN`。
   - 将服务器内部数据库地址 `nacp-mysql:3306` 替换为本地可访问的 `nacp.m.srl:3306`。
   - 只清理本地前后端端口，不再启动或停止本地 MySQL。
4. 更新 API 驱动测试规范，明确本地开发和测试站共用测试站 MySQL。

## 决策记录

`nacp.m.srl` 是专用测试站。为了减少本地库、测试站库、Docker volume 之间的维护成本，本地开发测试直接使用测试站数据库。

如果本地代码修改涉及数据库结构，可以直接让本地后端迁移/修改测试站数据库。迁移后测试站线上程序可能暂时因旧代码不兼容新结构而异常；这种情况接受，完成本地验证后部署新程序镜像即可恢复代码和数据库结构一致。

## 验证

1. `bash -n gogogo.sh` 通过。
2. 脚本不会在日志中输出数据库密码，只显示脱敏 DSN。

# NACP 日志 IP 记录与可见性策略

**日期**：2026-05-18  
**范围**：消费日志、错误日志、SFT 容错链路日志、用户日志查询、令牌日志查询、后续日志导出

## 结论

NACP 的 IP 策略调整为：

- **存储层默认记录 IP**：用户调用模型产生的消费日志、错误日志、SFT 步骤日志与摘要日志，应默认把 `ClientIP()` 写入 `logs.ip`。
- **用户可见层按设置脱敏**：用户个人设置中的 `record_ip_log` 不再表示“是否记录 IP”，而表示“用户端是否显示 IP”。
- **默认不可见**：默认 `record_ip_log=false`，用户在自己的日志、令牌日志、未来日志导出等用户可见位置看不到 IP。
- **管理员审计可见**：管理员日志查询、后台风控、统计与安全审计可以读取数据库内真实 IP。

## 设计原因

原版 NewAPI 的语义是用户开启后才记录 IP。NACP 调整为默认记录，是为了满足安全规范与反滥用需求：

- 恶意用户不能通过关闭个人设置阻止平台记录来源 IP。
- 管理员在追踪异常调用、滥用、攻击、撞库、批量注册、渠道异常时有完整审计线索。
- 用户端仍然保留隐私体验：用户不想看到 IP 时，前端和 API 不展示。

## 实现边界

### 写入路径

`model.RecordErrorLogWithType` 与 `model.RecordConsumeLog` 写入日志时直接记录：

```go
Ip: c.ClientIP()
```

不再读取 `RecordIpLog` 决定是否写入。

### 用户查询路径

用户可见日志必须经过脱敏：

- `/api/log/self`
- `/api/log/token`
- 后续如果增加用户侧日志导出，也必须走相同规则。

判断函数：

```go
model.CanUserViewLogIp(userId)
```

当返回 `false` 时，返回给用户前必须清空 `log.Ip`。

### 管理员查询路径

管理员接口保留真实 IP：

- `/api/log/`
- `/api/log/grouped`
- `/api/log/trace`
- 后台统计、审计、风控相关接口

管理员侧不受用户 `record_ip_log` 影响。

## UI 文案约束

用户侧文案保持原版 NewAPI 表达，不在界面暴露内部安全审计语义：

- 标题：`记录请求与错误日志IP`
- 说明：`开启后，仅"消费"和"错误"日志将记录您的客户端IP地址`

注意：这是用户体验文案。NACP 内部实际语义已经调整为“默认记录，用户侧按设置显示/隐藏”。后续不要为了匹配内部语义改动用户侧文案。

## 后续要求

新增任何日志导出、用户端日志详情、用户端链路详情、API key 自查接口时，必须先判断接口使用者身份：

- 管理员：可返回真实 IP。
- 普通用户：仅在 `record_ip_log=true` 时返回真实 IP，否则返回空字符串或 `-`。
- 公开/匿名接口：不得返回 IP。

测试时至少覆盖：

1. 用户关闭显示 IP：数据库有 IP，用户接口返回空 IP。
2. 用户开启显示 IP：数据库有 IP，用户接口返回真实 IP。
3. 管理员查询：不受用户设置影响，始终返回数据库真实 IP。
4. 旧日志兼容：旧日志 `ip=''` 时正常显示为空，不做推断或补写。
